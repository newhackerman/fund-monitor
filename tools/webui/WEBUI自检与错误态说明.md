# Web UI 自检与错误态说明（中文）

## 这次新增的内容
### 1. 页面顶部增加“接口状态”
用于快速判断：
- `/api/status` 是否正常
- `/api/summary` 是否正常

如果接口没通，不再靠页面各个卡片零散表现来猜问题。

## 错误态策略
### 以前的问题
页面取不到值时，容易显示成：
- 0
- --

这会让人误以为是“真实值为 0”。

### 现在的策略
如果核心字段取不到，页面显示：
- **读取失败**

只有在接口明确返回真实数字时，才显示数字。

## 为什么这样改
例如：
- `watchlist_count = 0`
  - 可能真是 0
  - 也可能是接口没取到

在监控页这种场景里，宁可明确提示“读取失败”，也不要误导成真实 0。

## 建议排查顺序
如果页面不对，先看：
1. 顶部 `接口状态`
2. `运行状态 / 监控基金数 / PID / 日志更新时间` 是否能同时成立
3. 如果这几项都失败，优先怀疑：
   - 服务没启动
   - 访问到了旧端口/旧实例
   - 容器里仍是旧进程
